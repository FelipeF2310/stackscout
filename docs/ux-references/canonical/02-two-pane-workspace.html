What you’re building

A private document Q&A assistant (RAG over your own PDFs).

MVP goal

Ask a question → get a cited answer drawn from your documents.

In scope (v1)

  • Upload & parse PDFs
  • Embed + store vectors
  • Retrieve & cite answers

Tradeoffs & alternatives

Tradeoffs you’re accepting

  • Managed services (clerk, openai-sdk) add vendor cost as you scale.
  • qdrant self-hosted means you operate it — or swap for managed.

Alternatives worth knowing

  • Vector: pinecone (managed) · weaviate (self-host)
  • Auth: authjs (self-host, full control)

Design notes — what the real reference frames changed

Why this variant exists

The full ~80s recording (cursor-flow-reference) shows a conversation that builds a living artifact — not a stepped wizard. Frame 06: it narrates a brief thought and announces “Switched to Plan Mode” with a reason. Frame 09: questions are an inline card in the thread (paginated, lettered A–F, Skip/Next). Frames 16–17: a two-pane view — conversation left, a Plan artifact right with In scope / Out of scope / an architecture diagram / behavior / next steps, plus a “Review Plan” card with a primary action.

Translated to StackScout (not copied)

Left = a calm conversation: your idea, a one-line reasoning beat, an announced Architecture Mode switch (our Plan Mode analog, capability-first rationale), reflected capabilities as removable chips, an inline A–E question card, then a Review brief card. Right = the living Architecture Brief from your DESIGN_INSPO — What you’re building, MVP goal, In/Out of scope, capability→tool, an architecture diagram (capability-first node map), tradeoffs (one tap), next step. No IDE chrome, file tree, terminal, or Cursor colors/type.

How it stays simple

Each pane does exactly one job (talk · the artifact). Questions live in the thread so there’s no separate wizard. Secondary depth (tradeoffs/alternatives) is one disclosure. Accent only on links, mode, and the active answer; near-black only on primary actions.

vs simplified.html

simplified.html = the same content as a calm linear stepped flow (good for narrow screens / lowest cognitive load). conversational.html (this) = the reference’s conversation→living-artifact rhythm, momentum preserved. Same StackScout content and tokens; pick the interaction model.

Still rough

One real question wired (Next just reveals the Review card + flips the artifact state to “ready”). Composer, model selector, add/remove chips, and tool links are visual. All data illustrative; deterministic/seed framing, no LLM/auth/persistence.
